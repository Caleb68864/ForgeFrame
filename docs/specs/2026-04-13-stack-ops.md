## Outcome
Three MCP tools â€” `effects_copy`, `effects_paste`, `effect_reorder` â€” enable reusing and reordering filter stacks on Kdenlive clips. `effects_copy` returns a portable stack dict; `effects_paste` applies it to another clip in `append` | `prepend` | `replace` modes; `effect_reorder` moves a filter within a clip's stack by index. Stateless caller-managed clipboard (no hidden server state). Same-project scope only.

## Intent
**Trade-off hierarchy (highest first):**
1. Statelessness over ergonomics â€” no hidden server/workspace state
2. LLM-friendly portability â€” stack dict is JSON-serializable, round-trippable
3. Additive changes to `patcher.py` â€” do not refactor Spec 1 additions
4. Primitive over wrapper â€” positional `(from_index, to_index)` now; `move_to_top` etc. deferred to Spec 7

**Preferences:**
- Prefer reusing `_iter_clip_filters` (private helper from Spec 1) rather than duplicating XML traversal
- Prefer strict JSON schema validation on paste input (reject malformed stacks loudly)
- Prefer rewriting `track=`/`clip_index=` attributes on paste rather than requiring the caller to sanitize

**Escalation triggers:**
- Any change to the serializer output format
- Any change to the `OpaqueElement.xml_string` contract
- Any change to existing `effect_add`, keyframe tools, or `_iter_clip_filters` signatures

## Context
Spec 1 (Keyframes) shipped three patcher methods (`get_effect_property`, `set_effect_property`, `list_effects`) and a private `_iter_clip_filters` helper at `workshop-video-brain/src/workshop_video_brain/edit_mcp/adapters/kdenlive/patcher.py`. ## Requirements

1. `effects_copy(workspace_path, project_file, track, clip)` returns a JSON-serializable stack dict of the target clip's filter stack.
2. `effects_paste(workspace_path, project_file, track, clip, stack, mode="append")` writes filters from a previously copied stack onto the target clip. Modes: `append` (default), `prepend`, `replace`.
3. `effect_reorder(workspace_path, project_file, track, clip, from_index, to_index)` moves a filter within the target clip's stack.
4. Paste rewrites `track=`/`clip_index=` XML attributes on each filter to match the target clip.
5. Paste preserves keyframe animation strings verbatim (no timestamp rewriting in v1).
6. All three tools auto-snapshot per existing policy and return `snapshot_id`.
7. Empty-stack copy and empty-stack paste succeed as no-ops with clear return data.
8. Out-of-range reorder indices return an `_err` envelope including current stack length.
9. Operations are same-project scope only; cross-project copy is explicitly out of scope.
10. Full test suite passes with zero regressions.

## Edge Cases

- **Empty-stack copy** â€” returns `effect_count: 0` with `effects: []`; not an error.
- **Empty-stack paste** â€” no-op, returns `effects_pasted: 0` with a note in the response.
- **Paste to same clip** â€” allowed. In `append` mode duplicates the stack; in `replace` mode is a no-op effectively (clear-then-reinsert-same).
- **Reorder `from_index == to_index`** â€” no-op, success with note.
- **Reorder out-of-range** â€” `IndexError` from patcher wrapped in `_err` listing current stack length.
- **Malformed stack JSON on paste** â€” `json.JSONDecodeError` wrapped in `_err` with pointer to `effects_copy` as expected producer.
- **Stack dict missing `effects` key** â€” `ValueError` in pipeline wrapped in `_err`.
- **Stack filter xml missing `track=`/`clip_index=` attributes** â€” paste rewrites them anyway; no error.
- **Paste with invalid `mode`** â€” `ValueError` wrapped in `_err` listing `append`, `prepend`, `replace`.
- **Copy from non-existent clip** â€” `IndexError` from `_iter_clip_filters` wrapped in `_err`.

## Out of Scope

- Cross-project copy/paste (belongs to Spec 4: stack presets)
- Workspace- or session-persisted clipboard (explicitly stateless per design)
- Semantic reorder wrappers (`move_to_top`, `move_up`, `move_down`, `move_to_bottom`) â€” Spec 7
- Keyframe timestamp shift on paste (`shift_to_clip_start` option) â€” deferred; v1 preserves timestamps verbatim
- Selective-effect copy (copy subset of filters by index/name) â€” current primitive copies all; a future wrapper can filter
- Undo via any mechanism other than the existing snapshot/restore machinery

## Constraints

### Musts
- All acceptance criteria above.
- Python 3.12+.
- Use the Spec 1 `_iter_clip_filters` helper for read paths (no duplicate XML traversal).
- Auto-snapshot on every write call.
- Preserve keyframe animation strings byte-exact through copy/paste.

### Must-Nots
- Must NOT introduce a workspace-level or session-level clipboard file.
- Must NOT modify existing Spec 1 methods or `_iter_clip_filters`.
- Must NOT change the serializer output for non-mutated filters.
- Must NOT mutate `OpaqueElement.xml_string` of filters that aren't the target of the current operation.

### Preferences
- Prefer additive changes to `patcher.py`.
- Prefer strict validation on paste input over permissive recovery.
- Prefer `_ok`/`_err` response envelopes over raising exceptions out of MCP tools.

### Escalation Triggers
- If `_iter_clip_filters` doesn't expose enough information to compute absolute insert positions, stop and ask.
- If `after_tractor` placement slot semantics differ from what's assumed (causing inserted filters to land in the wrong position), stop and ask.
- Any test fixture requirement beyond reusing the Spec 1 `keyframe_project.kdenlive` fixture.
