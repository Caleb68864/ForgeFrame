# Scenario MD-10: VideoProject, RenderJob, SnapshotRecord construction and serialization

## Description
Verify all models in `core/models/project.py` -- `VideoProject`, `RenderJob`,
`SnapshotRecord` -- construct correctly, that required fields are enforced,
that `ProjectStatus` and `JobStatus` enums are stored as strings, that
datetime fields default to UTC-aware timestamps, and that all models
round-trip through JSON and YAML.

## Preconditions
- Source module exists and is importable: `workshop_video_brain.core.models.project`

## Test Cases
- **TestVideoProjectRequired**: constructing without `title` raises `ValidationError`
- **TestVideoProjectDefaults**: `slug=""`, `status="idea"`, `content_type=""`
- **TestVideoProjectIdAutoGenerated**: two instances have different `id` UUIDs
- **TestVideoProjectCreatedAtUtc**: `created_at` is timezone-aware UTC
- **TestVideoProjectUpdatedAtUtc**: `updated_at` is timezone-aware UTC
- **TestVideoProjectStatusEnum**: `status` stored as plain string `"idea"`, not `ProjectStatus` instance
- **TestVideoProjectStatusTransition**: setting `status="published"` stored as `"published"`
- **TestVideoProjectInvalidStatus**: unknown status string raises `ValidationError`
- **TestVideoProjectAllFields**: all fields set explicitly; `model_dump()` matches
- **TestVideoProjectJsonRoundTrip**: `VideoProject.from_json(vp.to_json())` produces equal instance
- **TestVideoProjectYamlRoundTrip**: `VideoProject.from_yaml(vp.to_yaml())` produces equal instance
- **TestRenderJobRequired**: constructing without `workspace_id` or `project_path` raises `ValidationError`
- **TestRenderJobDefaults**: `profile=""`, `output_path=""`, `mode=""`, `status="queued"`, `started_at=None`, `completed_at=None`, `log_path=""`
- **TestRenderJobStatusEnum**: `status` stored as plain string `"queued"`
- **TestRenderJobStartedAtNone**: `started_at=None` survives `model_dump()` as `None`
- **TestRenderJobTimestampsSet**: passing ISO datetime strings for `started_at` and `completed_at` produces timezone-aware datetimes
- **TestRenderJobJsonRoundTrip**: `RenderJob.from_json(rj.to_json())` produces equal instance
- **TestSnapshotRecordRequired**: constructing without `workspace_id` raises `ValidationError`
- **TestSnapshotRecordDefaults**: `project_file_path=""`, `manifest_snapshot={}`, `description=""`
- **TestSnapshotRecordTimestampUtc**: `timestamp` is timezone-aware UTC
- **TestSnapshotRecordManifestSnapshot**: arbitrary dict survives JSON round-trip
- **TestSnapshotRecordYamlRoundTrip**: `SnapshotRecord.from_yaml(sr.to_yaml())` produces equal instance

## Steps
1. Read source module: `workshop_video_brain/core/models/project.py`
2. Create `tests/unit/test_project_model.py`
3. Implement all test cases
4. Run: `uv run pytest tests/unit/test_project_model.py -v`

## Expected Results
- `title` (VideoProject) and `workspace_id` + `project_path` (RenderJob) and `workspace_id` (SnapshotRecord) are required
- Status enums stored as plain strings via `use_enum_values=True`
- `None` optional datetime fields appear as `None` in `model_dump()`
- All datetime defaults are timezone-aware UTC
- Full JSON and YAML round-trip fidelity

## Pass / Fail Criteria
- Pass: All construction, validation, and serialization tests pass
- Fail: Any test fails
