# Scenario MD-05: MediaAsset field defaults, aliases, and enum handling

## Description
Verify `MediaAsset` (from `core/models/media.py`) constructs correctly with only
the required `path` field, that numeric/status defaults are correct, that the
MCP-alias pairs (`duration` / `duration_seconds`, `file_size` / `file_size_bytes`)
are both stored independently, that `use_enum_values=True` flattens all three
status enums to strings, and that the model round-trips through JSON and YAML.

## Preconditions
- Source module exists and is importable: `workshop_video_brain.core.models.media`

## Test Cases
- **TestMediaAssetRequired**: constructing without `path` raises `ValidationError`
- **TestMediaAssetDefaults**: with only `path`, all numeric fields default to `0` or `0.0`, `hash=""`, `proxy_path=""`
- **TestMediaAssetStatusDefaults**: `proxy_status="not_needed"`, `transcript_status="pending"`, `analysis_status="pending"` (stored as strings due to `use_enum_values=True`)
- **TestMediaAssetIdAutoGenerated**: two instances have different `id` UUIDs
- **TestMediaAssetCreatedAtUtc**: `created_at` is timezone-aware and set to UTC
- **TestMediaAssetDurationAlias**: setting `duration=5.0` stores that value; `duration_seconds` is independent (still `0.0` unless explicitly set)
- **TestMediaAssetFileSizeAlias**: same independence test for `file_size` / `file_size_bytes`
- **TestMediaAssetBothAliasesSet**: setting both `duration=5.0` and `duration_seconds=5.0` is valid and both survive `model_dump()`
- **TestMediaAssetColorFieldsNone**: `color_space=None`, `color_primaries=None`, `color_transfer=None` by default
- **TestMediaAssetColorFieldsSet**: passing string values for color fields stores and round-trips correctly
- **TestMediaAssetIsVfr**: `is_vfr=False` by default; setting `True` survives serialization
- **TestMediaAssetAllFields**: construct with every field explicitly set; `model_dump()` matches
- **TestMediaAssetJsonRoundTrip**: `MediaAsset.from_json(asset.to_json())` produces equal instance
- **TestMediaAssetYamlRoundTrip**: `MediaAsset.from_yaml(asset.to_yaml())` produces equal instance
- **TestMediaAssetInvalidProxyStatus**: invalid status string raises `ValidationError`
- **TestMediaAssetInvalidType**: passing `path=123` (int) raises `ValidationError`

## Steps
1. Read source module: `workshop_video_brain/core/models/media.py`
2. Create `tests/unit/test_media_model.py`
3. Implement all test cases
4. Run: `uv run pytest tests/unit/test_media_model.py -v`

## Expected Results
- `path` is the only required field; all others have documented defaults
- `duration` and `duration_seconds` are separate independent fields (no aliasing at Pydantic level)
- Status fields stored as plain strings (`"not_needed"`, `"pending"`, etc.)
- `created_at` is timezone-aware (`tzinfo` is not `None`)
- Full field round-trip through JSON and YAML without data loss

## Pass / Fail Criteria
- Pass: All construction, validation, and serialization tests pass
- Fail: Any test fails
