# Scenario MD-16: Workspace construction, nested VideoProject, and serialization

## Description
Verify `Workspace` (from `core/models/workspace.py`) constructs correctly with
its required fields (`project`, `media_root`, `workspace_root`), that the nested
`VideoProject` is validated on construction, that optional fields default
correctly, and that the full model -- including the nested project -- round-trips
through JSON and YAML without data loss.

## Preconditions
- Source module exists and is importable: `workshop_video_brain.core.models.workspace`
- `workshop_video_brain.core.models.project.VideoProject` is importable

## Test Cases
- **TestWorkspaceRequired**: constructing without `project`, `media_root`, or `workspace_root` raises `ValidationError` for each missing field
- **TestWorkspaceDefaults**: `vault_note_path=""`, `config={}`
- **TestWorkspaceIdAutoGenerated**: two `Workspace` instances have different `id` UUIDs
- **TestWorkspaceNestedProject**: `workspace.project` is a `VideoProject` instance with its own `id` and `title`
- **TestWorkspaceNestedProjectEnumFlattened**: `workspace.project.status` is a plain string (`"idea"`) not a `ProjectStatus` instance
- **TestWorkspaceMediaRootString**: `media_root` accepts an absolute path string and stores it verbatim
- **TestWorkspaceWorkspaceRootString**: `workspace_root` accepts an absolute path string and stores it verbatim
- **TestWorkspaceVaultNotePath**: non-empty `vault_note_path` stored and round-trips correctly
- **TestWorkspaceConfigDict**: arbitrary `config` dict with nested values survives JSON round-trip
- **TestWorkspaceModelDump**: `workspace.model_dump()` contains `project` key with a nested dict (not a `VideoProject` instance)
- **TestWorkspaceJsonRoundTrip**: `Workspace.from_json(ws.to_json())` produces equal instance including nested project fields
- **TestWorkspaceYamlRoundTrip**: `Workspace.from_yaml(ws.to_yaml())` produces equal instance
- **TestWorkspaceInvalidProject**: passing a plain dict without `title` as `project` raises `ValidationError`
- **TestWorkspaceProjectAcceptsDictCoercion**: passing a valid dict `{"title": "My Project"}` as `project` is coerced to `VideoProject` by Pydantic v2
- **TestWorkspaceMutableDefaultIsolation**: two `Workspace` instances do not share the same `config` dict object

## Steps
1. Read source module: `workshop_video_brain/core/models/workspace.py`
2. Create `tests/unit/test_workspace_model.py`
3. Define a helper fixture that builds a minimal valid `VideoProject` for reuse across tests
4. Implement all test cases
5. Run: `uv run pytest tests/unit/test_workspace_model.py -v`

## Expected Results
- `project`, `media_root`, and `workspace_root` are all required
- Auto-generated UUIDs differ across instances
- `workspace.project` is fully validated as a `VideoProject` on construction
- Pydantic v2 coerces a valid `dict` supplied as `project` to a `VideoProject` instance
- `model_dump()` serializes `project` as a nested dict
- Full JSON and YAML round-trips preserve all fields including nested project data
- `config` dict is isolated between instances

## Pass / Fail Criteria
- Pass: All construction, validation, and serialization tests pass
- Fail: Any test fails
