# Scenario MD-04: Marker, MarkerConfig, MarkerGroup, MarkerRule construction and serialization

## Description
Verify all models in `core/models/markers.py` -- `Marker`, `MarkerGroup`,
`MarkerRule`, `MarkerConfig` -- construct correctly, that `use_enum_values=True`
causes `MarkerCategory` enum instances to be stored as plain strings, and that
all models round-trip through JSON and YAML without loss.

## Preconditions
- Source module exists and is importable: `workshop_video_brain.core.models.markers`

## Test Cases
- **TestMarkerDefaults**: `confidence_score=0.0`, `source_method=""`, `reason=""`, `clip_ref=""`, `start_seconds=0.0`, `end_seconds=0.0`, `suggested_label=""`
- **TestMarkerIdAutoGenerated**: two `Marker` instances have different `id` values
- **TestMarkerCategoryRequired**: constructing `Marker` without `category` raises `ValidationError`
- **TestMarkerEnumStoredAsString**: `marker.category` is a `str`, not a `MarkerCategory` instance, due to `use_enum_values=True`
- **TestMarkerAllFields**: pass all fields; `model_dump()` returns matching dict
- **TestMarkerJsonRoundTrip**: `Marker.from_json(marker.to_json())` produces equal instance
- **TestMarkerYamlRoundTrip**: `Marker.from_yaml(marker.to_yaml())` produces equal instance
- **TestMarkerGroupDefaults**: `markers=[]`, `source=""`
- **TestMarkerGroupCategoryRequired**: constructing without `category` raises `ValidationError`
- **TestMarkerGroupWithMarkers**: list of `Marker` objects preserved through JSON round-trip
- **TestMarkerRuleRequired**: constructing without `keywords`, `category`, or `base_confidence` raises `ValidationError`
- **TestMarkerRuleAllFields**: all three required fields round-trip correctly
- **TestMarkerConfigDefaults**: `rules=[]`, `category_weights={}`, `silence_threshold_seconds=2.0`, `segment_merge_gap_seconds=3.0`
- **TestMarkerConfigWithRules**: rules list populated; JSON round-trip preserves rule count and content
- **TestMarkerConfigCategoryWeights**: `category_weights` dict with float values survives serialization
- **TestMarkerInvalidCategory**: passing an unknown string as `category` raises `ValidationError`
- **TestMarkerConfidenceBoundary**: `confidence_score=1.0` and `confidence_score=0.0` both accepted (no range validator in source -- confirm no error)

## Steps
1. Read source module: `workshop_video_brain/core/models/markers.py`
2. Create `tests/unit/test_markers_model.py`
3. Implement all test cases
4. Run: `uv run pytest tests/unit/test_markers_model.py -v`

## Expected Results
- `use_enum_values=True` means `marker.category` equals `"intro_candidate"`, not `MarkerCategory.intro_candidate`
- Auto-generated UUIDs differ across instances
- All required-field absences raise `pydantic.ValidationError`
- Float and list fields survive JSON and YAML round-trips without type coercion errors

## Pass / Fail Criteria
- Pass: All construction, validation, and serialization tests pass
- Fail: Any test fails
